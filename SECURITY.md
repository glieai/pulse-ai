# Security Policy

## Reporting a Vulnerability

If you discover a security vulnerability in Pulse, please report it responsibly:

**Email:** security@glie.ai

Please include:
- Description of the vulnerability
- Steps to reproduce
- Potential impact
- Suggested fix (if any)

## Response Timeline

- **Acknowledgment:** Within 48 hours
- **Initial assessment:** Within 1 week
- **Fix timeline:** Depends on severity, typically within 30 days

## Scope

This policy applies to the latest release of Pulse. Please ensure you're testing against the most recent version.

## Out of Scope

- Vulnerabilities in dependencies (report to the upstream project)
- Self-hosted instances with misconfigured environments
- Social engineering attacks

## Recognition

We appreciate security researchers who help keep Pulse safe. With your permission, we'll acknowledge your contribution in our release notes.
